---

| 項目 | 記入欄 |
|---|---|
| 日付 | 例: 2026-04-20 |
| モジュール | 例: Rapid Response Drill |
| シナリオ | 例: 急な進捗報告（経営層向け・30秒） |
| 制限時間 | 例: 30秒 |
| プロバイダ | 例: OpenAI / Fallback |
| Mode | 例: Remote / Fallback |

## Baseline 関連（Baseline Assessor 使用時のみ）

| 項目 | 記入欄 |
|---|---|
| 自己認識平均 | 例: 3.2 / 5.0 |
| AI 総合スコア | 例: 3.8 / 5.0 |

## 自己レビュー

**最初の一言はどうだったか**
> 例: 「進捗ですが、」から始めてしまい、結論が出てくるまで3文かかった。次は「結論から言うと、〜です」で始める。

**論点は散らなかったか**
> 例: 原因の説明に入りすぎて、次アクションを言えなかった。

**詰められたとき（または終盤のfollow-up）で崩れた箇所**
> 例: 「なぜ事前に気づかなかったのか」に対し、言い訳が先に出た。事実と解釈を分けて答えるべきだった。

**次回の改善点（1つだけ）**
> 例: 一言目を「結論から言うと」で固定する。

---

## AI レビュー要約

| 項目 | 記入欄 |
|---|---|
| 総合スコア | 例: 3.5 / 5.0 |
| 強みとして指摘されたこと | 例: 事実と解釈の分離が明確だった |
| リスクとして指摘されたこと | 例: 一言目が結論でなく状況説明になっていた |
| 次回アクション（AI提案） | 例: 30秒以内に結論→現状→次アクションの3点を揃える練習を優先する |

---

## 次回に向けた一文（最重要）

> 例: 次は必ず「結論から言うと」で口を開く。それだけを意識する。
