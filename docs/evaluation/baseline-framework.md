# Baseline Framework — 評価の観点と指標の意味

> **「採点結果をどう読むか」に迷ったらここを開いてください。** 評価の4層、各メトリクスの定義、スコアの解釈方法をまとめています。

- 最終更新: 2026-04-19

---

## 評価の考え方

ExplainerCore の評価は「一回のスコアで判断する」のではなく、**時間をかけて変化を追う**ことを目的にしています。

そのため、初回は完璧な採点よりも「後から比較できる一貫した基準を作ること」を優先してください。

---

## 評価の4層

採点は以下の4つの層で行います。どれか1つだけが高くても、実際の会話では通用しないことが多いためです。

| 層 | 見ていること |
|---|---|
| **内容** | 結論先行、構造化、要点充足、論点逸脱の少なさ |
| **話し方** | 長ポーズ、フィラー（「えっと」「あの」）、話速、抑揚の単調さ |
| **対話** | 応答速度、確認表現、follow-up への適応、割り込み回避 |
| **自己認識** | 自己評価とAI評価の差分、差分の縮小傾向 |

---

## 2段階の評価設計

### 1. Baseline診断（初回 + 2週間ごと）

現状の実力を多角的に把握します。

- 自己評価アンケート（スライダーで4能力を自己採点）
- AI面接官による模擬質問への音声回答
- 文字起こしの内容評価（AI採点）
- 4能力ごとの5段階スコア
- **自己認識平均とAI評価平均の差分（`selfPerceptionGap`）**

### 2. 継続評価（各セッション）

毎回の訓練セッションで記録します。

- 回答準備時間（`answerPreparationSec`）
- 回答の構造明確性
- 端的さ（制限時間内に締めまで届いたか）
- 論点逸脱率
- 反論への耐性
- 説得完了率

---

## メトリクス定義

| メトリクス名 | 意味 |
|---|---|
| `answerPreparationSec` | シナリオ提示後、録音開始するまでの準備時間（秒） |
| `timeLimitSec` | そのセッションで設定した制限時間（秒） |
| `charCount` | 文字起こし（transcript）の文字数 |
| `estimatedReadingSec` | transcript を読むのにかかる推定秒数 |
| `structureMarkerCount` | 結論・理由・次アクション等の構造マーカーの数 |
| `benefitMarkerCount` | 便益・効果に関する表現の数 |
| `repairMarkerCount` | 是正・次アクションに関する表現の数 |
| `hedgeMarkerCount` | 不確実性の明示に関する表現の数（「〜と思われます」等） |
| `closingMarkerCount` | 合意・次の一手に関する表現の数 |
| `fillerCount` | 「えっと」「あの」などのフィラーの出現数 |
| `longPauseCount` | transcript 上で検知できた長ポーズの数 |
| `selfPerceptionGap` | 自己評価とAI評価の差分（正 = AI評価が高い、負 = 自己評価が高い） |

## 4つの能力ごとの評価観点

### わかりやすさ
- 結論が先に出ているか
- 主張と理由が分かれているか
- 相手に必要な粒度で話せているか

### 即答力
- 質問を受けてからの反応速度
- 最初の一文が崩れていないか
- 30/60/90秒の時間制約内で要点を出せるか
- 分からない点を短く明示して前に進められるか

### 詰め対応
- 反論に対して論点を維持できるか
- 不明点を曖昧にごまかしていないか
- 説明責任と不要な自己否定を分けられるか
- 守った後に「再着地」まで示せるか

### 説得力
- 相手目線の便益が先に出ているか
- 反対意見への備えがあるか
- メリットとリスクを両面提示できるか
- 利害関係者に合わせて説明を調整できるか
- 最後に意思決定を前に進めているか

## スコアの読み方

| 状況 | 解釈と対応 |
|---|---|
| `selfPerceptionGap` が大きくプラス | AI評価 > 自己評価 → 実力を控えめに見積もっている可能性がある |
| `selfPerceptionGap` が大きくマイナス | 自己評価 > AI評価 → 実際の構造化や一言目の精度を見直す |
| 単回スコアが高い/低い | **最低3セッションの平均で判断する**。1回では判断しない |

---

## 保存される情報

採点結果には以下が保存されます。

**`baselineSelfCheck`（Baseline 自己診断）**
- 各質問への回答値
- 自己認識メモ（フリーテキスト）
- 自己認識平均と能力別平均

**`selfCheckComparison`（自己評価とAI評価の比較）**
- 自己認識平均
- AI総合スコア
- 差分と解釈コメント

---

## 運用ルール

- 初回診断は全モジュールを最低1回ずつ実施する
- 同じ採点フォーマットで **2週間ごとに再診断**する
- AI採点だけでなく、**自己レビュー3行も必ず残す**
- Baseline Assessor では自己認識メモを残し、AI評価との差分も確認する
