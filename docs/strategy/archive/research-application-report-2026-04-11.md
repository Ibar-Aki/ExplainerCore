## エグゼクティブサマリー

今回の調査結果から見て、`ExplainerCore` は能力分類そのものは妥当であり、今回の更新で評価軸と訓練設計もかなり研究準拠に近づいた。今後の主課題は、内容構造だけでなく、話し方、対話ダイナミクス、自己認識差分の推移まで継続的に扱うことである。

最も優先すべきなのは、新しい機能を増やすことではなく、「何を良い応答と定義するか」を研究準拠で明確化することである。具体的には、`data/rubrics` を先に拡張し、30 / 60 / 90 秒の時間設計、長ポーズやフィラー、不確実性の扱い、利害関係者別説明、反論先回り、自己認識差分を正式な評価対象に入れるべきである。

その次に、`data/scenarios` を難度別、相手別、圧の種類別に増やし、`docs/evaluation` と `docs/training` を新評価軸に合わせて更新するのが順当である。`coach-web` の UI やログ設計は、その後で評価軸に従って拡張する進め方がもっとも破綻しにくい。

## 調査結果から見えた重要論点

### 1. 評価は単一総合点より観点分解が有効

調査結果では、説明力や会話力を単一スコアで扱うより、「内容」「話し方」「対話」「自己認識」のように分解した方が、改善要因を特定しやすく、訓練にも結びつけやすかった。現在の `ExplainerCore` でも構造化や相手視点は見ているが、まだ音声・会話ダイナミクスの扱いが弱い。

### 2. 即答力は速度だけでなく時間設計が重要

即答力の調査では、単に素早く話すよりも、30 秒、60 秒、90 秒で「何を削り、何を残すか」を切り替えられる能力が重要だった。現状の Rapid Response は方向性として正しいが、秒数別の設計や負荷段階の明示が不足している。

### 3. 詰め対応は防御だけでなく再着地が必要

圧の強い場面では、事実と解釈の分離、不確実性の明示、責任の取り方、不要な自己否定の回避が重要だった。さらに重要なのは、ただ守るだけでなく「何を確認し、どう着地させるか」を示す再着地力である。現状の Pressure Defense はこの方向に向いているが、境界線管理や不当な詰めへの対応はまだ薄い。

### 4. 説得は相手の意思決定設計として捉えるべき

説得と合意形成の調査では、自分の主張を通すことより、相手が意思決定しやすい状態を設計することが重要だった。現在の Persuasion Lab は相手視点とクロージングを見ているが、利害関係者別の説明最適化、両面提示、想定反論の先回りは今より強く扱うべきである。

### 5. 継続運用は回数より反復の質が重要

想起練習、自己説明、If-Then、視点取得、低速呼吸、間隔反復などは、短時間で回しやすく、継続しやすい訓練法として再利用価値が高かった。現状の warmup 設計は良い出発点だが、「今日の弱点に応じてどれを選ぶか」までルール化した方が学習効率が上がる。

## 現状との差分

### 現在の強み

- 4能力の切り分けは妥当
- Baseline で自己認識差分を扱っている
- 各モジュールに推奨型と warmup の発想がある
- Session Review Dashboard の方向性は継続学習と相性がよい

### 現在の不足

- 会話ダイナミクス指標が不足している
- フィラー、長ポーズ、話速、抑揚などの扱いが弱い
- 30 / 60 / 90 秒の時間制約設計が rubric と scenario に落ちていない
- 詰め対応で境界線管理と再着地力が弱い
- 説得で利害別最適化と両面提示が弱い
- 週間運用が間隔反復より回数消化に寄っている

## 推奨事項

### 優先度 P0

#### 1. ルーブリックを研究準拠に拡張する

最優先は `data/rubrics` の更新である。評価観点が固定されると、シナリオ、UI、ダッシュボードの更新方針も揃う。

- `baseline-assessment-rubric.json`
  - `metacognition` を正式観点に追加する
  - `interaction` を追加し、確認表現や対話の自然さを見る
  - `aggregateMetrics` に `longPauseCount` `fillerCount` `selfPerceptionGap` を入れる前提で定義を整える
- `rapid-response-rubric.json`
  - 30 / 60 / 90 秒で答え切る力を観点化する
  - 不確実性を短く明示して前に進める力を入れる
- `pressure-defense-rubric.json`
  - 責任の取り方と不要な自己否定回避を分けて見る
  - 再着地の重みを今より上げる
- `persuasion-rubric.json`
  - 利害関係者別の説明調整を追加する
  - 両面提示と反論先回りを明示的に評価する

#### 2. シナリオを難度別に増やす

既存シナリオは良いが、パターンが少なく反復で慣れやすい。研究結果に沿って、負荷段階と相手違いを追加するべきである。

- Rapid Response
  - 30 秒、60 秒、90 秒の3段階を追加する
  - 情報不足、経営層向け、オンライン会議向けを増やす
- Pressure Defense
  - 責任追及、事実確認、感情的圧力を分ける
  - 不当な詰めに対する境界線維持ケースを加える
- Persuasion
  - 上司、現場責任者、他部署調整先など相手別ケースを追加する
  - リスクが重い提案を加え、両面提示を鍛える
- Baseline
  - 短い状況報告、抽象説明、優先順位変更など説明タイプを分ける

#### 3. warmup の選択ルールを固定する

研究結果をそのまま warmup 設計へ落とすべきである。

- わかりやすい説明
  - 自己説明
  - 抽象と具体の往復
  - 聞き手別リライト
- 即答力
  - 30 秒想起練習
  - If-Then 即答プラン
  - 30 / 60 / 90 秒切り替え
- 詰め対応
  - 低速呼吸
  - 事実と解釈の分離
  - 境界線つき再着地
- 説得力
  - 視点取得
  - 両面提示
  - 反論先回り

### 優先度 P1

#### 4. セッション記録を拡張する

将来の記録項目は、内容評価だけでなく、会話ダイナミクスの追跡まで含めるべきである。

- `responseLatencyMs`
- `longPauseCount`
- `fillerCount`
- `selfPerceptionGap`
- `followUpHandlingScore`

#### 5. Dashboard を自己比較中心に強化する

継続利用の観点では、他人比較より自己比較が重要である。したがって Dashboard は次の表示を優先するべきである。

- 直近3回平均との差分
- 自己評価と AI 評価の差分推移
- 能力別改善率
- 崩れやすいポイントの自動要約

#### 6. 週間運用を間隔反復に寄せる

同一能力を連日続けるより、1日置きに再測定する運用の方が研究結果と整合しやすい。

- 月曜: Baseline or 説明
- 火曜: 即答
- 水曜: 説得
- 木曜: 詰め対応
- 金曜: 即答か説明の再測定

### 優先度 P2

#### 7. ユーモア会話は補助テーマとして扱う

ユーモアの調査結果は有益だが、現時点では本流モジュール化よりも補助訓練で扱う方が適切である。

- warmup の補助テーマとして扱う
- ラポール形成の一言目テンプレートに限定反映する
- rubric の中核観点には混ぜすぎない

## 推奨する実行順序

1. `data/rubrics` を更新する
2. `data/scenarios` を増やす
3. `docs/evaluation` と `docs/training` を新観点に合わせて更新する
4. その後に `coach-web` の記録項目と UI 表示を更新する

## 結論

今回の調査結果を反映するうえで最も重要なのは、「まず評価軸を固め、その上で訓練と UI を揃える」ことである。`ExplainerCore` はすでに方向性は良いので、次に必要なのは横に機能を広げることではなく、研究に沿って評価精度と訓練再現性を上げることである。
