## 目的

このプロジェクトは、仕事で求められる説明力と会話力を、感覚論ではなく再現可能な訓練として整備するためのものです。特に、会議で急に話を振られたときの即答力、厳しい追及を受けたときの防御力、相手を納得させる説得力を重点対象にします。

## 対象者

- 日本語で社内会議や報告を行うビジネスパーソン
- 会議での説明に苦手意識があるが、訓練で改善したい人
- 説明や提案を、その場で整理して伝える必要がある人

## 成功条件

- 4 能力ごとに明確な評価軸がある
- 週単位で訓練を回せる
- 訓練結果が履歴として残り、比較できる
- 自己認識と AI 評価の差分が見える
- 説明テンプレートと切り返しパターンが蓄積される
- 使い続けても記録が散逸しない

## 重要原則

- まず日本語の社内会議・報告に絞る
- 音声とテキストの両方を扱う
- AI 採点に依存しすぎず、自己レビューと履歴比較も残す
- UI/UX 変更は必ず文書に反映する
- 学習素材よりも、実戦に近い状況再現を優先する
- provider 設定状態と実行結果を混同しない

## 範囲

含むもの:
- 調査プロンプト
- 採点ルーブリック
- 訓練シナリオ
- ローカル Web アプリ
- セッション記録
- 振り返りテンプレート

含まないもの:
- 企業向けのマルチユーザー運用
- クラウド同期
- 一般英会話の訓練
- 営業 CRM や会議システムとの連携

## 成果物の最小単位

- `docs`: 意思決定と運用の説明
- `prompts`: 外部 LLM へ投げる命令文
- `data`: 実装が読む JSON 定義
- `apps/coach-web`: 実行可能な訓練 UI
- `sessions`: 実際の訓練ログ
- `templates`: 手動レビュー補助

## 判断基準

機能追加や変更時は、以下の順で判断します。

1. 実戦の訓練に直接効くか
2. 継続運用時に記録と比較がしやすいか
3. 実装とドキュメントの整合が保てるか
4. ローカル環境で無理なく回るか
