## 目的

調査結果をもとに、`ExplainerCore` の既存アプリ改善、既存評価方法改善、新規アプリ提案を分けて整理し、今後のプロダクト判断材料にする。

## 2026-04-12 実装反映メモ

このレポートで優先度高としていた改善のうち、今回の更新では以下まで反映済みです。

- `Session Setup` に 30 / 60 / 90 秒プリセットを追加
- `Scenario` に `successSignals` と `constraints` の先出しを追加
- `Evaluation` に制限時間、推定発話時間、フィラー数、長ポーズ記号数、自己認識差分などの簡易メトリクスを追加
- `Session Review Dashboard` に自己認識差分の平均カードを追加

一方で、まだこの文書の提案段階に残っているのは以下です。

- 自己評価と AI 評価の差分推移を折れ線などで可視化すること
- transcript 冒頭に特化した一言目レビュー導線
- Pressure Defense と Persuasion の専用 UI 分岐

## エグゼクティブサマリー

調査結果から見ると、`ExplainerCore` は「説明力を訓練する基盤」として良い出発点にある。今回の更新で、自己認識差分の平均、時間枠切り替え、簡易メトリクス表示までは入ったが、会話ダイナミクスの本格記録、自己認識差分の推移可視化、相手別説明最適化の専用 UI はまだ今後の課題である。

改善方針は3つに分かれる。第一に、既存アプリは `Session Review Dashboard` を中心に、自己比較と崩れポイントの可視化を強化するべきである。第二に、既存評価方法は、内容・話し方・対話・自己認識の4層へ再設計するべきである。第三に、新規アプリや新規モジュールは、いきなり大きく増やすより、既存モジュールの隙間を埋める形で、短時間訓練と事前練習を支えるものから追加するべきである。

## 1. 既存アプリ改善提案

### 1-1. Session Review Dashboard を主導線に格上げする

現在の情報設計では、Dashboard はレビュー置き場として設計されているが、調査結果を踏まえると、継続利用の中核としてもっと強く位置付けるべきである。

推奨改善:

- `直近3回平均との差分` を最上段に置く
- `自己評価と AI 評価の差分推移` を折れ線で見せる
- `長ポーズ` `フィラー` `一言目` `論点逸脱` などの崩れポイントを自動要約する
- 次回の推奨 warmup と推奨シナリオを、弱点から直接提案する

期待効果:

- 自己比較中心の継続導線になる
- 単発採点から継続改善へ重心が移る
- 次に何を練習すべきかが明確になる

### 1-2. Practice Workspace に「時間枠切り替え」を入れる

即答力と説明力の研究では、30 秒、60 秒、90 秒で何を残すかを切り替える訓練が有効だった。したがって、セッション開始時に時間枠を選び、同じ問いを複数時間で練習できる UI が有効である。

推奨改善:

- `Session Setup` に 30 / 60 / 90 秒の選択を追加する
- 回答後に「同じ内容を 30 秒短縮版で再回答する」導線を追加する
- 制限時間ごとの評価結果を横並び比較できるようにする

### 1-3. 録音から評価までの間に「一言目レビュー」を挿入する

調査結果では、特に即答と詰め対応では最初の一文の質が大きかった。現状は回答全体の評価に寄っているので、冒頭の数秒に特化した振り返り UI を持つ価値がある。

推奨改善:

- transcript の最初の 1 文だけを抜き出して表示する
- `結論になっているか` `保留条件になっているか` `言い訳で始まっていないか` を短く評価する
- 良い一言目の言い換え候補を1つだけ返す

### 1-4. 圧の高いモジュールは見た目と導線を分ける

Pressure Defense は認知負荷が高いので、他モジュールと同じ導線だと疲れやすい。圧の強い場面では、事実整理と呼吸リセットの導線を最初に出した方がよい。

推奨改善:

- 開始前に `事実 / 解釈 / 未確認` の3分割メモを置く
- 低速呼吸の 20 秒ガイドを即表示できるようにする
- follow-up を 1 問ずつ表示して、同時に圧をかけすぎない

### 1-5. 説得モジュールは相手タイプ切り替えを前面に出す

説得の研究では、同じ内容でも相手によって説明の構造を変える必要があった。現状の Persuasion Lab も改善余地が大きい。

推奨改善:

- `上司` `現場責任者` `他部署` `経営層` を選べるようにする
- UI 上に `相手の関心軸` を常時表示する
- 提案後に `相手の懸念` を返して 2 ターン目を練習できるようにする

## 2. 既存評価方法改善提案

### 2-1. 評価を4層へ再設計する

既存評価は主に構造化と初動速度を見ているが、研究結果では4層で持つ方が改善要因を切り分けやすい。

- 内容
  - 結論先行
  - 構造化
  - 要点充足
  - 論点逸脱の少なさ
- 話し方
  - 長ポーズ
  - フィラー
  - 話速
  - 抑揚の単調さ
- 対話
  - 応答遅延
  - 確認表現
  - follow-up への適応
  - 割り込み回避
- 自己認識
  - 自己評価と AI 評価の差分
  - 差分の縮小傾向

### 2-2. 総合点より観点別の改善率を重視する

ユーザーが知りたいのは「今日は何点だったか」より「前回よりどこが良くなったか」である。したがって、評価設計は総合点中心から観点別改善率中心に寄せるべきである。

推奨改善:

- モジュールごとに `今回伸びた観点` と `次に崩れやすい観点` を返す
- `総合点` は要約として残し、主表示は `観点別差分` にする
- Baseline は `自己認識差分` を主要カードにする

### 2-3. 時間制約別の期待値を分ける

30 秒回答と 90 秒回答を同じ rubric で見ると評価がぶれやすい。したがって、秒数別の期待値が必要である。

推奨改善:

- 30 秒は `結論 / 現状 / 次アクション` を中心に見る
- 60 秒は `理由` や `不確実性明示` まで見る
- 90 秒は `トレードオフ` や `利害関係者別調整` まで見る

### 2-4. 詰め対応では「守り」と「再着地」を分けて見る

圧の強い場面では、単に崩れなかっただけでは不十分である。守り切ったかと、その後に会話を前へ進めたかは別物なので、評価も分けるべきである。

### 2-5. 説得では「相手が動ける状態になったか」を見る

説得評価は話し手の上手さより、相手が判断できる状態になったかで見る方が研究結果と一致する。

推奨改善:

- 相手の便益が明示されたか
- リスクが隠されずに扱われたか
- 反論が先回りされていたか
- 次の意思決定が明確だったか

## 3. 新規アプリ提案

### 3-1. Quick Reframe Coach

目的:

短時間で説明を組み直す練習に特化した補助アプリ。長い説明を 30 秒、60 秒、90 秒へ圧縮し直す。

主な機能:

- 長い transcript を入力して時間別に再構成する
- 一言目の言い換え候補を返す
- `結論先行になっているか` をチェックする

向いている理由:

今回の研究では、即答力と説明力の共通基盤として「時間制約下での構造化」が重要だったため。

### 3-2. Follow-up Defense Trainer

目的:

詰め対応の中でも、1問目ではなく follow-up で崩れる問題に特化する。

主な機能:

- 最初の回答に対して連続 follow-up を返す
- `事実 / 解釈 / 未確認` のどこで崩れたかを表示する
- 境界線を保つ返し方を提示する

向いている理由:

実務では初回回答より追撃質問で崩れやすく、現行アプリの補完になるため。

### 3-3. Stakeholder Persuasion Mapper

目的:

同じ提案内容を相手別に作り分ける訓練に特化する。

主な機能:

- 同じ提案を `上司` `現場` `他部署` `経営層` 向けに並べて比較する
- 相手ごとに `便益` `懸念` `判断材料` を自動整理する
- 両面提示の不足を指摘する

向いている理由:

説得研究で最重要だった「相手が意思決定しやすい設計」を直接鍛えられるため。

### 3-4. Meta-Cognition Tracker

目的:

自己評価と AI 評価の差分推移に特化した軽量アプリ。

主な機能:

- セッション前の自己採点
- セッション後の AI 評価との比較
- 過大評価 / 過小評価の傾向表示

向いている理由:

自己認識差分は今回の研究で再発見された重要テーマであり、既存アプリの横断補助として独立価値があるため。

## 優先順位

### すぐやるべきこと

- Dashboard 強化
- 時間枠切り替え UI
- 4層評価への再設計

### 次にやるべきこと

- Pressure Defense の専用導線
- Persuasion の相手タイプ切り替え
- 観点別改善率表示

### 余力があればやること

- Quick Reframe Coach
- Follow-up Defense Trainer
- Stakeholder Persuasion Mapper
- Meta-Cognition Tracker

## 結論

調査結果をもとにしたプロダクト改善では、「既存アプリの評価と継続導線を強くすること」が最も費用対効果が高い。新規アプリは有望だが、まずは `ExplainerCore` 本体で、時間制約、follow-up、自己認識差分、相手別説得を扱えるようにした上で、補助アプリへ切り出すのが妥当である。
