## 能力と訓練モジュールの対応

| 能力 | 主担当モジュール | 補助モジュール | 狙い |
| --- | --- | --- | --- |
| わかりやすい説明 | Baseline Assessor | Session Review Dashboard | 現状の話し方の癖と説明構造を把握する |
| 即答力 | Rapid Response Drill | Session Review Dashboard | 制限時間内で結論から話す習慣を作る |
| 詰め対応 | Pressure Defense Simulator | Rapid Response Drill | 追及に崩されず論点を維持する |
| 説得力 | Persuasion Lab | Session Review Dashboard | 相手の意思決定を前に進める説明を鍛える |

## トレーニングの基本サイクル

1. モジュールに対応した `Evidence-Based Warmup` を 1 つ選ぶ
2. シナリオ選択
3. 回答準備
4. 口頭回答
5. 録音と文字起こし
6. AI 採点
7. 自己レビュー
8. 次回の改善ポイント設定

## モジュール別の推奨ウォームアップ

### Baseline Assessor

- 自己説明プロンプト
- 具体観点つきセルフレビュー
- 聞き手別リライト

### Rapid Response Drill

- 30秒想起練習
- If-Then 即答プラン
- 30 / 60 / 90 秒切り替え
- 分散リピート設計

### Pressure Defense Simulator

- 低速呼吸リセット
- 詰め対応 If-Then
- 再着地の自己説明
- 事実 / 解釈 / 未確認 の切り分け

### Persuasion Lab

- 視点取得プロンプト
- 提案の自己説明
- 反論対応 If-Then
- 両面提示の一言目練習

## モジュールごとの型

### Baseline Assessor

- 目的: 現状把握
- 推奨型: `結論 -> 理由 -> 影響 -> 次の打ち手`
- 重点: 自己認識と実際の出力のズレ確認

### Rapid Response Drill

- 目的: 初動速度と端的さ
- 推奨型: `結論 -> 今の状況 -> リスク / 不確実性 -> 次アクション`
- 重点: 一言目、無音短縮、時間内完結、30 / 60 / 90 秒の切り替え

### Pressure Defense Simulator

- 目的: 論点維持と守り方
- 推奨型: `事実 -> 現時点の解釈 -> 不足情報 -> 是正策`
- 重点: 事実と解釈の分離、感情に引っ張られないこと、再着地と境界線管理

### Persuasion Lab

- 目的: 納得感のある提案
- 推奨型: `相手の利益 -> 提案内容 -> 懸念への対応 -> 合意したい次ステップ`
- 重点: 相手視点、反論先回り、両面提示、クロージング

## 週間運用例

- 月曜: Baseline 再確認 1 本
- 火曜: 即答ドリル 2 本
- 水曜: 説得シナリオ 2 本
- 木曜: 詰め対応 2 本
- 金曜: 即答か説明の再測定と履歴レビュー

## 訓練の強化ポイント

- 即答時は「結論 -> 理由 -> 次アクション」の型を徹底する
- 即答時は 30 / 60 / 90 秒で削る情報を変える
- 詰め対応では「事実 -> 解釈 -> 次の確認事項」で守り、最後に再着地する
- 説得では「相手の優先事項 -> 提案の利得 -> リスクの透明化 -> 次の一手」を使う
- ウォームアップは 1 回で詰め込まず、その日の重点に合うものを 1 つだけ選ぶ

## 訓練結果の反映先

- transcript の傾向は `sessions/transcripts`
- 採点結果は `sessions/reviews`
- 重要な手動学びは `templates/session-review-template.md` を使って整理する

## 改善サイクル

1. 弱いモジュールを履歴で特定する
2. そのモジュールを 2 本続けて回す
3. transcript を見て一言目の改善案を作る
4. 次の週に同じ能力を別シナリオで再測定する
