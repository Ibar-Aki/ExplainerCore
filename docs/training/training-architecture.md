## 能力と訓練モジュールの対応

| 能力 | 主担当モジュール | 補助モジュール | 狙い |
| --- | --- | --- | --- |
| わかりやすい説明 | Baseline Assessor | Session Review Dashboard | 現状の話し方の癖と説明構造を把握する |
| 即答力 | Rapid Response Drill | Session Review Dashboard | 制限時間内で結論から話す習慣を作る |
| 詰め対応 | Pressure Defense Simulator | Rapid Response Drill | 追及に崩されず論点を維持する |
| 説得力 | Persuasion Lab | Session Review Dashboard | 相手の意思決定を前に進める説明を鍛える |

## トレーニングの基本サイクル

1. シナリオ選択
2. 回答準備
3. 口頭回答
4. 録音と文字起こし
5. AI 採点
6. 自己レビュー
7. 次回の改善ポイント設定

## モジュールごとの型

### Baseline Assessor

- 目的: 現状把握
- 推奨型: `結論 -> 理由 -> 影響 -> 次の打ち手`
- 重点: 自己認識と実際の出力のズレ確認

### Rapid Response Drill

- 目的: 初動速度と端的さ
- 推奨型: `結論 -> 今の状況 -> リスク -> 次アクション`
- 重点: 一言目、無音短縮、時間内完結

### Pressure Defense Simulator

- 目的: 論点維持と守り方
- 推奨型: `事実 -> 現時点の解釈 -> 不足情報 -> 是正策`
- 重点: 事実と解釈の分離、感情に引っ張られないこと

### Persuasion Lab

- 目的: 納得感のある提案
- 推奨型: `相手の利益 -> 提案内容 -> 懸念への対応 -> 合意したい次ステップ`
- 重点: 相手視点、反論先回り、クロージング

## 週間運用例

- 月曜: Baseline 再確認 1 本
- 火曜: 即答ドリル 3 本
- 水曜: 詰め対応 2 本
- 木曜: 説得シナリオ 2 本
- 金曜: 履歴レビューと改善点整理

## 訓練の強化ポイント

- 即答時は「結論 -> 理由 -> 次アクション」の型を徹底する
- 詰め対応では「事実 -> 解釈 -> 次の確認事項」で守る
- 説得では「相手の優先事項 -> 提案の利得 -> 不安への先回り」を使う

## 訓練結果の反映先

- transcript の傾向は `sessions/transcripts`
- 採点結果は `sessions/reviews`
- 重要な手動学びは `templates/session-review-template.md` を使って整理する

## 改善サイクル

1. 弱いモジュールを履歴で特定する
2. そのモジュールを 2 本続けて回す
3. transcript を見て一言目の改善案を作る
4. 次の週に同じ能力を別シナリオで再測定する
