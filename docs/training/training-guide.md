# Training Guide — 訓練の設計と日次・週次の回し方

> **「今日何をやるか」に迷ったらここを開いてください。** 訓練の仕組み・モジュールの使い方・日次の最小メニュー・週次レビューをまとめています。

- 最終更新: 2026-04-19

---

## 4つの能力とモジュールの対応

ExplainerCore は4つの能力を訓練します。各能力には担当モジュールがあります。

| 能力 | 担当モジュール | 補助モジュール | 目標 |
|---|---|---|---|
| わかりやすい説明 | Baseline Assessor | Session Review Dashboard | 自分の話し方の癖と説明構造を把握する |
| 即答力 | Rapid Response Drill | Session Review Dashboard | 制限時間内に「結論から話す」習慣を作る |
| 詰め対応 | Pressure Defense Simulator | Rapid Response Drill | 追及に崩されず論点を維持する |
| 説得力 | Persuasion Lab | Session Review Dashboard | 相手の意思決定を前に進める説明を鍛える |

---

## トレーニングの基本サイクル

1セッションの流れは以下です。

```
① 今日の重点能力を1つ決める
② 対応モジュールの Evidence-Based Warmup を1本選んで実行
③ シナリオを選択
④ 口頭で回答（録音）
⑤ 文字起こしを入力
⑥ AI 採点を実行
⑦ 自己レビューを3行書く
⑧ 次回の改善ポイントを1つ決める
```

## 各モジュールの推奨型とウォームアップ

### Baseline Assessor — 現状把握
**推奨型:** `結論 → 理由 → 影響 → 次の打ち手`  
**重点:** 自己評価とAI評価のズレを確認する

**ウォームアップ（1つ選ぶ）:**
- 自己説明プロンプト（60秒）— 結論・根拠・次の一手の3問に答える
- 具体観点つきセルフレビュー — 過去の録音を「一言目」「構造」「次アクション」の3点で見直す
- 聞き手別リライト — 同じ内容を上司向け/現場向けで言い換える

---

### Rapid Response Drill — 即答力
**推奨型:** `結論 → 今の状況 → リスク/不確実性 → 次アクション`  
**重点:** 一言目を崩さない、時間内に締める、30/60/90秒で削る情報を変える

**ウォームアップ（1つ選ぶ）:**
- 30秒想起練習 — 型を見ずに30秒で話す
- If-Then 即答プラン — 「もし急に振られたら、最初にこう言う」を3本作る
- 30/60/90秒切り替え — 同じ内容を時間枠に合わせて省略する練習

---

### Pressure Defense Simulator — 詰め対応
**推奨型:** `事実 → 現時点の解釈 → 不足情報 → 是正策`  
**重点:** 事実と解釈を分ける、感情に引っ張られない、守った後に再着地する

**ウォームアップ（1つ選ぶ）:**
- 低速呼吸リセット（3分）— 4秒吸って6秒吐く×10〜15回
- 詰め対応 If-Then — 「もし原因を詰められたら、事実と推測を分けて答える」を3本作る
- 再着地の自己説明 — 守り切った後に「何を確認して、どう着地するか」を声に出す

---

### Persuasion Lab — 説得力
**推奨型:** `相手の利益 → 提案内容 → 懸念への対応 → 合意したい次ステップ`  
**重点:** 相手目線の便益を先に出す、反論を先回りする、両面提示する

**ウォームアップ（1つ選ぶ）:**
- 視点取得プロンプト（3分）— 相手の関心・損失回避・判断しにくい理由を書き出す
- 提案の自己説明 — 相手の言葉で提案を一言にする
- 反論対応 If-Then — 「もし反対されたら、相手の懸念を先に要約する」を作る

## 日次の最小メニュー

### 15分コース（最低限）

```
5分:  ウォームアップ 1本
10分: モジュール 1セッション
```

### 30分コース（推奨）

```
5分:  ウォームアップ 1本
20分: モジュール 2セッション（うち1本は時間枠か相手条件を1段変える）
5分:  振り返り（自己レビュー3行 + 次回の改善ポイント）
```

### セッション後の確認チェックリスト

```
□ 一言目で結論を言えたか
□ 理由と影響が分離されていたか
□ 不明点を断定せずに明示できたか
□ 制限時間内に締めまで届いたか
□ follow-up のどこかで崩れた場合、どこでか
□ 次アクションを言えたか
□ transcript と自己感覚がズレていたか
```

## 週次の運用

### 間隔反復スケジュール（推奨）

同じ能力を毎日固定で回すより、**1日置きに別の能力を鍛え、週末に再測定する**ほうが定着しやすいです。

| 曜日 | やること |
|---|---|
| 月曜 | Baseline 再確認 / 説明力セッション 1本 |
| 火曜 | 即答ドリル 2本 |
| 水曜 | 説得シナリオ 2本 |
| 木曜 | 詰め対応 2本 |
| 金曜 | 即答か説明の再測定 + 履歴レビュー |

### 週次レビュー（金曜または週末）

`Session Review Dashboard` を開いて以下を確認します。

1. 弱いモジュールはどれか
2. 自己評価とAI評価のズレが大きかったのはどのセッションか
3. 前週より改善した能力・悪化した能力は何か
4. 来週の重点能力を2つまでに絞る
5. 追加したいシナリオや修正したいルーブリックをメモする

---

## Baseline の回し方

- **初回は必ず実施する**（現在地を確認するため）
- 以後は **2週間に1回程度** で十分
- 自己評価とAI評価の差分（`selfPerceptionGap`）が大きいときは、スコアだけ追わず transcript を読む
- **差分が大きい = 実力の過大/過小評価** のどちらかなので、一言目の実際の出力と自己感覚のズレを先に見る

---

## 改善サイクル

```
① Session Review Dashboard で弱いモジュールを確認する
② そのモジュールを2本続けて回す
③ transcript を見て「一言目」の改善案を作る
④ 次の週、同じ能力を別シナリオで再測定する
```
